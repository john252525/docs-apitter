CRM integration

Get Amocrm pipelines

Retrieve pipelines from Amocrm. Requires Amocrm URL. This endpoint provides the list of pipelines that can be used by the user to select which pipeline should be used for creating an AMOCRM source. All passed urls will be treated as Amocrm urls.

Authorizations:
RefererAuthApiKeyAuth
query Parameters
crm_url
required
string
Example: crm_url=integrilla.amocrm.ru

CRM URL (only amocrm urls - integrilla.amocrm.ru)

header Parameters
X-Api-Key
required
string
Example: apikey
Referer
required
string
Example: https://client.domain.com/

Responses

Binder API production server

https://bapi88.apitter.com/api/v1/int/getAmocrmPipelines

Request samples

  • cURL Example
curl -X POST "https://bapi88.apitter.com/api/v1/int/getAmocrmPipelines" \
-H "Content-Type: application/json" \
-H "Accept: application/json" \
-H "X-API-Key: secret_key" \
-H "Referer: https://somedomain.com/" \
-d '{
  "crm_url": "integrilla.amocrm.ru",
}'

Connect CRM with messaging platform

Previously available by binder call: POST https://b2288.apitter.com/403*****-****-****-****-3be******712?keykey=adminkey&json Initializes and creates integration between a CRM system and a messager vendor (channel). Automatically identifies CRM type from the provided parameters if possible and creates necessary vendor records and binds, adds webhook and Amocrm source.

Authorizations:
RefererAuthApiKeyAuth
Request Body schema: application/json
required
touchapi_source
required
string

Touchapi account source (e.g., whatsapp, telegram)

touchapi_token
required
string

Token for the messaging platform vendor

touchapi_login
required
string

Login for the messaging platform vendor

crm_url
required
string

CRM domain URL. Supported formats:

  • domain.amocrm.ru
  • domain.Kommo.com
  • domain.bitrix24.ru
  • domain.bitrix24.com
crm_type
string

Required when type cannot be derived from url

crm_token
string

Crm token data

crm_data
object

Additional papams specific for CRM system. Amocrm - Pipeline ID optional for accounts with single pipeline.

touchapi_main
boolean

Special flag for Amocrm integration indicating main instance. When true, clears touchapi_main flag from other instances.

touchapi_name
string

Custom name for the integration (used in Amocrm source naming)

touchapi_phone
string

Phone number associated with the integration

Get CRM integration status

Returns the integration status for a specific CRM by its URL. The status indicates the current step of integration process. Works correct only for newly added account. case UNAVAILABLE = 0; case DISABLED = 1; case NOT_INSTALLED = 2; case INSTALLED = 5;

Authorizations:
RefererAuthApiKeyAuth
query Parameters
crm_url
required
string
Example: crm_url=integrilla.amocrm.ru

CRM URL to check status for (e.g., domain.amocrm.ru, domain.Kommo.com, domain.bitrix24.ru) Required parameter

Get vendors

Returns a list of vendors. Can be filtered by login or use pagination. through limit and offset parameters. Either login or both limit/offset must be provided. Critical changes: Receiving all vendors now is unavailable. Recieving all vendors at once can be dangerous causing very heavy responses.

Authorizations:
RefererAuthApiKeyAuth
query Parameters
login
string
Example: login=login777

Filter by vendor login. Required when limit and offset are not provided

limit
integer [ 1 .. 1000 ]
Example: limit=10

Maximum number of records to return Required when login is not provided

offset
integer >= 0
